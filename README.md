# Window OCR Monitor

一个面向 Windows 的通用窗口 OCR 监控工具。

它会监控你选择的桌面应用窗口，基于屏幕截图 + OCR 检测窗口中的文本变化，并在识别到新的监控内容后执行：

- 声音提醒
- 桌面通知
- 整窗截图与 ROI 裁图留档

## 功能概览

- 手动截图校准
- 手动框选 `question_roi` 和 `status_roi`
- 基于 Windows OCR 的离线文字识别
- 基于状态机的内容去重，避免同一内容反复提醒
- 保存/加载本地 JSON 配置
- 导出当前调试截图
- 可自定义状态关键词
- 提供通用关键词示例 preset
- PyInstaller 打包 spec

## 运行方式

1. 进入项目目录：

```powershell
cd <repo-dir>
```

2. 安装依赖。

3. 启动程序：

```powershell
python main.py
```

## 首次使用

1. 打开需要监控的目标窗口，并确保界面处于可见状态。
2. 在工具里刷新窗口列表并选择目标窗口。
3. 点击“截图校准”。
4. 分别框选：
   - `question_roi`：你希望重点 OCR 的主内容区域
   - `status_roi`：按钮、状态、倒计时、标签等辅助判定区域
5. 按需调整采样间隔、稳定帧阈值、变化阈值、截图目录和声音文件。
6. 根据实际场景修改“状态关键词 / Preset”中的示例关键词。
7. 点击“开始监控”。

## 默认示例配置 / Preset

项目内置的是通用示例关键词，而不是某个特定平台的专用配置。默认关键词更适合作为起点，你可以根据自己的监控场景改成更合适的词，例如：

- 开始
- 继续
- 确认
- 提交
- 保存
- 完成
- 下一步
- 通知
- 告警
- 剩余时间

示例配置文件见：

- `window_ocr_monitor.sample.json`

## OCR 依赖说明

本项目默认使用 Windows OCR（WinRT）进行离线识别。

如果启动监控时提示无法创建 OCR 引擎，请检查：

- 已安装 `pyproject.toml` 或 `requirements.txt` 中列出的 `winrt-*` 依赖
- Windows 已安装中文或英文 OCR 语言包
- 系统语言相关组件可正常使用

## 打包

在项目目录下执行：

```powershell
pyinstaller window_ocr_monitor.spec
```

## 说明

- 当前版本只做提醒与截图留档，不做自动点击或其他自动操作。
- 当目标窗口尺寸变化并与校准尺寸不一致时，程序会提示重新截图校准。
